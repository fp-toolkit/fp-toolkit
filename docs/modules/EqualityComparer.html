Module EqualityComparer

An EqualityComparer is intended to model deterministic or "decidable" equality between two values. Meaning, it should always give back the same result for the same inputs.

Generally you won't need to worry about these things, but in case you come up against some weird edge cases, EqualityComparers should always satisfy these rules:

  1. always return true if given two values that are the same object reference
    • myEqComparer.equals(a, a) === true
  2. always return true for equivalent variables, regardless of the order in which they are passed
    • myEqComparer.equals(a, b) === myEqComparer.equals(b, a)
  3. should be transitive such that if a = b and b = c, then a = c

Note: An EqualityComparer is structurally compatible with the Eq type from fp-ts.

Example

interface Pet {
   readonly name: string
   readonly age: number
}

class PetByNameComparer implements EqualityComparer<Pet> {
   equals(p1: Pet, p2: Pet): boolean {
       return p1.name === p2.name
   }
}
