Module Rec

A suite of functions for working with Record<> types in an immutable, functional way. The motivating use-case for using a Record<> over a Map<> is that Records are plain JavaScript objects, which means they are serializable by default. Because of objects' limitations, however, keys must extend the string type.

Important Note: The functions in this module ignore nullish values, because the type of V (values) must be non-nullish. For example, if you create a Rec<string, number> like { prop: undefined }, calculating Rec.size will return 0 and Rec.isEmpty will return true. This behavior aligns Rec more consistently with Map semantics.
